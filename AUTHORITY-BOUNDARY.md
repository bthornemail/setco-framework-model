# Authority Boundary

## Core rule

```
SETCO classifies.
OMI defines.
No layer may impersonate another.
```

## What SETCO may do

```
classify claims
categorize artifacts
stage proposals
adjudicate promotion eligibility
teach relationships
enforce authority boundaries
```

## What SETCO may not do

```
define OMI notation
claim OMI authority
validate OMI state
receipt OMI transitions
execute OMI runtime
replace OMI proof authority
```

## Layer boundaries

```
SETCO          = classification framework (outside-in)
OMI            = object model (inside-out)
omi-axioms     = proof authority rail
omi-isa        = executable artifact rail
Omnicron       = resolver surface
Metatron       = scribe/witness
Tetragrammatron = governor/validator
```

## Prohibited equivalences

```
Coq proof           != C implementation
Haskell type        != runtime behavior
TypeScript projection != validation
Python tool         != canon
omi-lisp declaration != application command
omi-port declaration != external connection
projection          != authority
```
