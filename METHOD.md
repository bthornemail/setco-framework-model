# Method

## Hypercube graph methodology

SETCO uses a hypercube graph as its classification workspace. The graph has
independent axes that can be expanded without collapsing prior distinctions.

### Axes

```
Epistemic:      KK / KU / UK / UU
Construction:   rule / fact / algorithm / proof / proposal
Adjudication:   P0 / P1 / P2 / P3 / P4
Implementation: proved / implemented / defined_model / interpretation
Projection:     impotent / innocuous / implementation-promoted
Interface:      omi-notation / omi-lisp / omi-port / Omi-* configuration
Dimensional:    point / line / plane / space / hyperspace
```

### Graph vocabulary

```
vertices  = classified claims or artifacts
edges     = typed transitions, dependencies, promotions, projections, lowerings
faces     = classification planes
cells     = integrated construction regions
hypercube = expandable methodology for adding independent axes
```

## Classification vocabulary

```
geometry:  node / edge / face
mode:      state / transform / phase
grammar:   noun / verb / adjective
```

Mnemonic boundary:

```
node   = state   = noun
edge   = transform = verb
face   = phase   = adjective/context
```

## Adjudication status ladder

```
P4  metaphor
P2  model
P1  finite check
P0  proof
```

## Promotion ladder

```
proposal
  -> defined model
  -> implementation precedent
  -> tested artifact
  -> proof target
  -> proved invariant
```

## Propagation chain

```
proved invariant
  -> canon name
  -> declaration surface
  -> implementation witness
  -> receipt
  -> projection
```
