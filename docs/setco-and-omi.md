# SETCO and OMI

> "SETCO classifies. OMI defines. Projection is not authority."

## Relationship

SETCO classifies how external things relate to the Omi Object Model. OMI defines the pure computational paradigm. They occupy distinct layers and must not be collapsed.

```text
SETCO:
  classification, categorization, canonization, adjudication
  promotion, propagation, pedagogy
  blackboard staging, canvas visualization, OS-like working surface

OMI:
  pure object/address model, declarative notation
  deterministic relation frame, bounded observer/projection logic
```

## The Layered Architecture

```text
External Environments (Coq, Haskell, TypeScript, C, Python, ...)
        ↓
SETCO (transition and classification framework)
        ↓
Omi Object Model (omi-notation, omi-lisp, omi-port)
        ↓
OMI Artifact Rails (omi-axioms, omi-isa, receipts)
```

SETCO sits above OMI. It is the bridge that lets existing tools approach the Omi Object Model without becoming its authority.

## Geometric Division

| Shape  | System                | Role                                      |
|--------|-----------------------|-------------------------------------------|
| Tetrahedron | omi-object-model | Pure relational faces (notation, lisp, port) |
| Cube   | setco-framework-model | Transition and classification workspace   |
| Sphere | Omnicron core         | Invariant deterministic binary core       |

## Doctrine

- SETCO does not define the object model.
- OMI does not classify how external tools relate.
- No layer may impersonate another.
- Projection is not authority.
