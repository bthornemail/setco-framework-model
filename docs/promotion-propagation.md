# Promotion and Propagation

> "SETCO classifies. OMI defines. Projection is not authority."

## Promotion

Promotion is the upward movement through the adjudication ladder:

```text
P4 metaphor → P2 model → P1 finite check → P0 proof
```

Or as construction types:

```text
proposal → defined model → implementation precedent
  → tested artifact → proof target → proved invariant
```

Each step requires:
- Known boundary
- Met requirements
- Declared authority limits
- Adjudication approval

## Propagation

Propagation is the outward flow of a proved or accepted claim through the system:

```text
proved invariant → canon name → declaration surface
  → implementation witness → receipt → projection
```

SETCO tracks the movement. OMI supplies the object model. The artifact rails supply proof and execution.

## Relationship

```text
Promotion:  moves claims upward in confidence (P4 → P0)
Propagation: moves claims outward through surfaces (proof → receipt → display)
```

SETCO manages both. Neither creates authority — they track and classify it.

## Boundary

SETCO stages and classifies promotion and propagation. The artifact either builds or it does not. The test either passes or it does not. SETCO does not make the runtime true.
