# Canonization and Adjudication

> "SETCO classifies. OMI defines. Projection is not authority."

## Adjudication Ladder (P0–P4)

```text
P4  metaphor       — aspirational, no formal standing
P3  proposal       — stated intent, no verification
P2  model          — defined structure, not yet checked
P1  finite check   — tested or compiled, not proved
P0  proof          — formally proved invariant
```

A claim moves up the ladder only when its boundary is known and its requirements are met.

## Promotion Path

```text
P4 metaphor → P3 proposal → P2 model → P1 finite check → P0 proof
```

Or equivalently:

```text
proposal → defined model → implementation precedent → tested artifact → proof target → proved invariant
```

## Canonization Process

1. **Intake**: External artifact arrives from Coq, Haskell, TypeScript, C, Python, etc.
2. **Classification**: What kind of object is this? (rule, fact, algorithm, proof, proposal)
3. **Boundary Declaration**: What does it support? Where does it fail? What authority does it lack?
4. **Status Assignment**: Which P-level does it currently hold?
5. **Adjudication**: Does it meet the requirements for its claimed level?
6. **Promotion**: If requirements met, advance to next level.
7. **Registration**: Record in the claim registry with its status and boundary.

## Authority Boundary

- Proof canonizes admissible structure.
- Validation determines accepted state.
- Attestation witnesses accepted transition.
- Projection displays accepted state.
- SETCO classifies what is allowed to count as what.
- SETCO does not decide truth by itself.
