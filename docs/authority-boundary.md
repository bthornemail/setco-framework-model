# Authority Boundary

> "SETCO classifies. OMI defines. Projection is not authority."

## Core Rule

No layer may impersonate another. Each system has a defined role. Authority is never transferred through classification — it is only declared.

## Authority Declarations

```text
Proof canonizes admissible structure.
Validation determines accepted state.
Attestation witnesses accepted transition.
Projection displays accepted state.
SETCO classifies what is allowed to count as what.
```

## Prohibited Equivalences

```text
A proposal is not a proof.
A proof is not an implementation.
An implementation is not a theorem.
A projection is not validation.
A metaphor is not authority.
An operating surface is not the object model.
A Coq proof ≠ C implementation.
A Haskell type ≠ runtime behavior.
A TypeScript projection ≠ validation.
A Python tool ≠ canon.
An omi-lisp declaration ≠ application command.
An omi-port declaration ≠ external connection.
```

## What SETCO Does Not Do

- SETCO does not decide truth.
- SETCO does not create authority.
- SETCO does not replace the proof rail.
- SETCO does not replace the implementation rail.
- SETCO does not replace the Omi Object Model.

## What SETCO Does

- Classifies what is allowed to count as what.
- Stages claims for adjudication.
- Tracks promotion and propagation.
- Prevents authority confusion across layers.

## Canonical Lock

```
SETCO classifies. OMI defines. Projection is not authority.
```
