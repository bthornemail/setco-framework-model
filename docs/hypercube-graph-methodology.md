# Hypercube Graph Methodology

> "SETCO classifies. OMI defines. Projection is not authority."

## Overview

SETCO uses a hypercube graph methodology for classifying, developing, and adjudicating claims before they are admitted into deterministic construction.

```text
Pyramid:         assurance ascent
Cube:            typed pedagogy
Tetrahedral frame: local Omi-* development face
Hypercube graph: full SETCO methodology
```

## Seven Axes

```text
1. Epistemic axis:       KK / KU / UK / UU
2. Construction axis:    rule / fact / algorithm / proof / proposal
3. Adjudication axis:    P0 / P1 / P2 / P3 / P4
4. Implementation axis:  proved / implemented / defined_model / interpretation
5. Projection axis:      impotent / innocuous / implementation-promoted
6. Interface axis:       omi-notation / omi-lisp / omi-port / Omi-* configuration
7. Dimensional axis:     point / line / plane / space / hyperspace
```

Each axis adds one independent direction of classification. The purpose: prevent category collapse.

## Graph Vocabulary

```text
vertices:  classified claims or artifacts
edges:     typed transitions, dependencies, promotions, projections, lowerings
faces:     classification planes (two-axis comparison surfaces)
cells:     integrated construction regions
hypercube: expandable methodology for adding independent axes
```

## Edge Types

```text
proposes, refines, implements, proves,
depends_on, projects_to, interfaces_with,
promotes_to, rejects_for_this_frame
```

Each edge is a typed transition, not just a line.

## Expansion Principle

The hypercube expands by adding new independent axes without collapsing old ones. Each new axis adds classification depth. The cube is the first complete pedagogical solid. The hypercube is the scalable research method.

```text
A proposal is not a proof.
A proof is not an implementation.
An implementation is not a theorem.
A projection is not authority.
A metaphor is not canon.
```

## What It Does Not Do

The hypercube does not prove the claim. It classifies the claim. Proof belongs to omi-axioms. Execution belongs to omi-isa. Classification belongs to SETCO.
