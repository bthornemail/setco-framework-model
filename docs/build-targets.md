# Build Targets

> "SETCO classifies. OMI defines. Projection is not authority."

## Two-Target Meta-Compilation

SETCO supports two distinct build targets:

### 1. SETCO Framework

```text
setco root framework
setco-blackboard (classification workspace)
setco-canvas (hypercube graph and visualization)
setco-os (IDE-like operating surface)
```

This target produces the transition framework itself: charter, methodology, classification grammar, canonization/adjudication model, public explanation, and cross-repo map.

### 2. OMI Object Model

```text
omi-notation
omi-lisp
omi-port
omi-axioms (proof authority rail)
omi-isa (executable artifact rail)
```

This target produces the pure computational paradigm: notation, declarations, proofs, implementation witnesses, receipts, and projections.

## Separation

SETCO classifies how external environments relate to OMI. OMI defines the object model. The two targets must not be collapsed. SETCO is not OMI. OMI is not SETCO.

## Doctrine

- SETCO classifies. OMI defines. Projection is not authority.
- No build target may impersonate another.
- The framework and the object model remain distinct.
