# Classification Grammar

> "SETCO classifies. OMI defines. Projection is not authority."

## Three Dimensions

SETCO classifies any idea, claim, object, artifact, or relation through three independent dimensions. Each dimension provides a handle before proof, implementation, projection, or interpretation are allowed to act.

## Geometry (Node / Edge / Face)

```text
Node (vertex):
  What is it? An object, claim, fact, rule, proof, proposal, artifact, or address.

Edge (link):
  How does it relate or move? A transition, dependency, promotion, propagation, or transform.

Face (graph):
  What surface or context makes it readable? A context, classification plane, projection, or interpretive frame.
```

## Mode (State / Transform / Phase)

```text
State:
  What remains stable? The object-position, identity-bearing checkpoint.

Transform:
  What movement or operation occurs? The relation, transition, delta.

Phase:
  What orientation or context is visible? The readable surface, classification frame.
```

## Grammar (Noun / Verb / Adjective)

```text
Noun:
  What is named? The object.

Verb:
  What acts, moves, relates, or changes? The operation.

Adjective:
  What qualifies or colors it? The qualified surface or context.
```

## Core Mnemonic

```text
Node  = State      = Noun
Edge  = Transform  = Verb
Face  = Phase      = Adjective / Context
```

## The Mnemonic Boundary

The purpose is not to reduce every idea to a toy diagram. The purpose is to prevent category collapse.

Before asking whether a concept is true, executable, canonical, or useful, SETCO asks where it belongs:

```text
Is it a node?   Is it state?   Is it noun?
Is it an edge?  Is it transform? Is it verb?
Is it a face?   Is it phase?   Is it adjective?
```

Only after that classification may the idea enter the blackboard, canvas, proof registry, implementation rail, or OMI declaration surface.
