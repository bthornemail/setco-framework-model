# Charter

## Root problem

Modern computational environments produce source code, type systems, proof
assistants, tests, runtime artifacts, OS surfaces, documents, diagrams,
protocols, interfaces, and human interpretations. Without a classification
framework, these collapse into false equivalences: a proof becomes equal to
an implementation, a projection becomes equal to validation, a notation
becomes equal to execution.

## Six principles

### 1. Distinction before authority

Classification precedes authority. Before anything can be promoted, it must
be classified.

### 2. Boundary before promotion

Each layer preserves its own authority boundary. No layer may impersonate
another.

### 3. OMI is the object model

The Omi Object Model is the pure computational paradigm. It owns notation,
declaration, address, and relation.

### 4. SETCO is the transition framework

SETCO classifies, stages, adjudicates, and teaches. It does not define the
object model.

### 5. Proof, implementation, and projection must remain separate

A proposal is not a proof. A proof is not an implementation. An
implementation is not a theorem. A projection is not validation.

### 6. SETCO does not create authority

SETCO classifies what is allowed to count as what. It does not decide truth
by itself.

## One-sentence lock

SETCO is the independent hypercube-graph transition framework that classifies,
stages, adjudicates, and teaches how modern computational environments may
loosely connect to the Omi Object Model without becoming its authority.
