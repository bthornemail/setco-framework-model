# Glossary

## SETCO

Systematic Epistemic Theological Computational Ontology. The outside-in
transition framework for classifying, staging, adjudicating, and teaching
how external computational environments relate to the Omi Object Model.

## OMI

Omicron Object Model. The inside-out pure object/address/relation model.
Owns notation, declaration, address, and relation.

## Blackboard

The SETCO classification and categorization adapter. External workspace
where claims, rules, facts, algorithms, proofs, proposals, code artifacts,
language bindings, and projection faces are staged before lowering into
pure OMI notation.

## Canvas

The SETCO visual graph surface. Makes classification visible as a
hypercube graph.

## Hypercube graph

SETCO's classification methodology. Independent axes that expand without
collapsing prior distinctions.

## Promotion

The process by which a classified claim advances through the adjudication
ladder: proposal -> defined model -> implementation precedent -> tested
artifact -> proof target -> proved invariant.

## Propagation

The process by which a proved invariant reaches external surfaces:
canon name -> declaration surface -> implementation witness -> receipt
-> projection.

## Adjudication

The process of determining which status level a claim occupies in the
promotion ladder.

## Non-collapse

The principle that each layer preserves its own authority boundary.
No layer may impersonate another. Projection is not authority.

## Authority boundary

The line that separates what one layer may do from what it may not.
SETCO classifies. OMI defines. Neither crosses the other's boundary.
